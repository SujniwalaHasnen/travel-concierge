[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of how a user request flows through our system.
First, the request enters the START node and immediately goes into the security checkpoint.
The security checkpoint scrubs high-risk PII like emails, phones, and passport numbers, and verifies that the prompt does not contain injection attacks or violate our budget cap.
If a security event is triggered, the flow is redirected to the security error node which blocks the session.
Otherwise, the clean query is forwarded to the main orchestrator agent.
The orchestrator agent delegates tasks to two sub-agents: travel_planner for itinerary building, and visa_assistant for document checklists.
Both sub-agents execute their tasks using a shared local MCP Server that provides flight, hotel, and visa checklist tools.
Once the sub-agent responses are synthesized by the orchestrator, the flow hits the review itinerary node where the system pauses for a human-in-the-loop approval.
If the traveler requests changes, it routes back to the orchestrator to revise the trip; otherwise, once approved, it outputs the finalized itinerary.
